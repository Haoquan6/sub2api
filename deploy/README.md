# PixelHub Docker Deployment

This directory contains the Docker Compose files used to deploy PixelHub with PostgreSQL and Redis.

## Files

| File | Description |
|------|-------------|
| `docker-compose.local.yml` | Recommended deployment file. Data is stored in local directories for easy backup and migration. |
| `docker-compose.yml` | Named-volume deployment file. Data is stored in Docker volumes. |
| `docker-deploy.sh` | One-click helper that creates `.env`, generates secrets, prepares data directories, and can start the stack. |
| `.env.example` | Environment variable template. |
| `Dockerfile` | Alternative Dockerfile copy kept for deploy-only workflows. |

## One-Click Deployment

From a cloned PixelHub repository:

```bash
git clone https://github.com/Haoquan6/sub2api.git PixelHub
cd PixelHub
bash deploy/docker-deploy.sh --start
```

The helper will:

- generate `deploy/.env` from `.env.example`
- generate `JWT_SECRET`, `TOTP_ENCRYPTION_KEY`, `POSTGRES_PASSWORD`, and `ADMIN_PASSWORD`
- create `deploy/data`, `deploy/postgres_data`, and `deploy/redis_data`
- build the local source into `pixelhub:latest`
- start PixelHub, PostgreSQL, and Redis

After startup, open:

```text
http://localhost:8080
```

The generated admin account is printed by the script and stored in `deploy/.env`.

## Manual Deployment

```bash
git clone https://github.com/Haoquan6/sub2api.git PixelHub
cd PixelHub/deploy
cp .env.example .env

# Edit required values before first start:
# POSTGRES_PASSWORD, JWT_SECRET, TOTP_ENCRYPTION_KEY, ADMIN_PASSWORD
nano .env

mkdir -p data postgres_data redis_data
docker compose -f docker-compose.local.yml up -d --build
```

Useful commands:

```bash
docker compose -f docker-compose.local.yml ps
docker compose -f docker-compose.local.yml logs -f pixelhub
docker compose -f docker-compose.local.yml restart pixelhub
docker compose -f docker-compose.local.yml down
```

## Compose File Choice

| File | Data Storage | Best For |
|------|--------------|----------|
| `docker-compose.local.yml` | `./data`, `./postgres_data`, `./redis_data` | Recommended. Easy backup and server migration. |
| `docker-compose.yml` | Docker named volumes | Simple local experiments or Docker-managed storage. |

## Upgrade

From the repository root:

```bash
git pull
docker compose -f deploy/docker-compose.local.yml up -d --build
```

## Backup And Migration

When using `docker-compose.local.yml`, the deployment can be migrated by archiving the repository directory after stopping the stack:

```bash
cd PixelHub/deploy
docker compose -f docker-compose.local.yml down

cd ..
tar czf pixelhub-complete.tar.gz PixelHub/
```

On the new server:

```bash
tar xzf pixelhub-complete.tar.gz
cd PixelHub/deploy
docker compose -f docker-compose.local.yml up -d --build
```

## Environment Variables

The most important variables are:

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `POSTGRES_PASSWORD` | Yes | - | PostgreSQL password. |
| `JWT_SECRET` | Recommended | auto-generated by app if empty | JWT signing secret. Set this explicitly for persistent sessions. |
| `TOTP_ENCRYPTION_KEY` | Recommended | auto-generated by app if empty | TOTP encryption key. Use `openssl rand -hex 32` and set this explicitly before using 2FA. |
| `SERVER_PORT` | No | `8080` | Host port exposed by PixelHub. |
| `ADMIN_EMAIL` | No | `admin@pixelhub.local` | Initial admin email. |
| `ADMIN_PASSWORD` | Recommended | auto-generated by app if empty | Initial admin password. |
| `POSTGRES_USER` | No | `pixelhub` | PostgreSQL user. |
| `POSTGRES_DB` | No | `pixelhub` | PostgreSQL database name. |
| `TZ` | No | `Asia/Shanghai` | Container timezone. |

See `.env.example` for the full configuration list.

## Notes

- PostgreSQL and Redis are included in the Compose stack.
- The application image is built from the local source by default.
- Keep `deploy/.env` private. It contains database passwords and signing secrets.
- If you need the data management daemon, see `DATAMANAGEMENTD_CN.md`; its socket path remains `/tmp/sub2api-datamanagement.sock` for compatibility with the current backend.
